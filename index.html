Project Overview

What is CSI?

Channel State Information (CSI) describes how wireless signals propagate from transmitter to receiver. Human movements alter these signal patterns, allowing us to detect activities without cameras or wearables.

Why This Project?

  • Contactless activity monitoring for smart homes/healthcare
  • Better privacy than camera-based systems
  • Low-cost compared to radar/UWB solutions
  • Works with existing WiFi infrastructure

CSI patterns for different human activities

Methodology

1. CSI Data Collection

Collected from Intel 5300 WiFi cards with 30 subcarriers. Dataset includes 6 activities: walking, running, sitting, standing, falling, and no activity.

2. Data Preprocessing

Noise removal with Butterworth filter, PCA for dimensionality reduction, and sequence segmentation into 5-second windows.

3. Hybrid CNN-LSTM Model

Combines CNN for spatial feature extraction and LSTM for temporal pattern recognition in CSI sequences.

Optimization Algorithms

Algorithm Type Key Parameters Advantages
SCA (Sine Cosine) Math-based r1, r2, r3, r4 Simple, fast convergence
GWO (Grey Wolf) Swarm-based Alpha, Beta, Delta Balanced exploration/exploitation
HHO (Harris Hawk) Nature-inspired Escape energy, Jump strength Adaptive search strategy

Implementation

Data Processing Pipeline

  1. Raw CSI data collection from WiFi NIC
  2. Noise reduction with 5Hz low-pass filter
  3. Amplitude/phase extraction from subcarriers
  4. Normalization (Min-Max scaling)
  5. Sequence segmentation (100 samples/window)
  6. Train-test split (80-20 ratio)

# Sample CSI Data Processing
import numpy as np
from scipy import signal

def process_csi(csi_data):
    # Butterworth low-pass filter
    b, a = signal.butter(4, 5, 'low', fs=100)
    filtered = signal.filtfilt(b, a, csi_data)
    
    # Min-Max normalization
    normalized = (filtered - np.min(filtered)) / (np.max(filtered) - np.min(filtered))
    
    return normalized

# Load dataset
csi_dataset = np.load('csi_data.npy')
processed = np.apply_along_axis(process_csi, 1, csi_dataset)

Key Findings

  • Optimal architecture: 2 CNN + 1 LSTM layers
  • Learning rate range: 0.001-0.005 performed best
